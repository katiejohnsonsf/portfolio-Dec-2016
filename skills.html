Skills App


  • Research
  • Design
  • Testing

I was at a growing development and design consultancy where the team was not familiar with each other's skills and experiences. This created a barrier to collaboration across the organization. The team wanted a way to solve this problem as the consultancy scaled, so we decided to create a Skills app.




stakeholder interviews: To get a fuller view of stakeholder goals for the organization, I interviewed each role on the team.






personas: Personas helped to clarify the goals for each stakeholder group related to knowledge sharing and skill development.






storyboards: I chose two scenarios from the stakeholder interviews. The scenarios also supported organizational priorities for the skills management tool.






team workshop: Using the personas, each team member took on a different role other than their own to perform the scenarios. The team created a task analysis and user flows from the scenarios they had acted out.


I used standard iOS elements to create an MVP with the user flows from the team workshop.


user flows: user flows: Using the personas and scenarios, each team member created their own task analysis and presented it to the group. We created a single set of user flows with stickies on the whiteboard.




site map: I combined the user flows into a site map to get a sense of the overall architecture of the product.



thumbnail flows with callouts: I created a visual guide to communicate the details of the design to stakeholders before engineers began the first build.




Prototype: I used Proto.io to build out the functionality of the interactive prototype.

Various team members provided feedback using the prototype of the design throughout the design process as they were the users.
